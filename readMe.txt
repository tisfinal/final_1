팀원: 재경,향기,성근,다혜,진하 

모든 소스 파일에 *시작 주석* 을 답니다. 

*시작 주석*: 밑에 있는거 복사해서 해당 소스 파일 클래스명 위에 붙여넣습니다.

/* 
 *  파일명 : 해당 소스 파일에 클래스명 혹은 소스파일명 을 적습니다.    ex => login.jsp
 * 
 *  설명 : 해당 소스파일의 역할을 파일명으로만 봐도 알수있겠지만, 간단한 설명또한 같이 적습니다.  ex=> login 폼태그가 있는 페이지 
 *
 *  create날짜 : 해당 소스 파일을 작성하기 시작한 날짜를 꼭 적습니다.  
 * 
 * update : 마지막 수정한 날짜를 적습니다. 
 *
 * 작성자 : 자기 이름을 혹은 같이 작업한 사람들의 이름 을 적습니다.  모르는부분을 작성자에게 묻기가 쉽습니다.  ex => 향기,진하
 */

주석을 달아주실 때 주의 사항.
1. 한 문장으로 주석이 끝날 경우 //
2. 여러 문장이 필요할 경우 /**/

필수로 주석작성해야하는 것들.  
전역변수, 함수, 다른페이지로 보내는 파라미터, 2번 이하 사용되는 함수를 생성이 아닌 사용하는 곳(자주사용하는 함수는 함수를 직접 찾아보지만, 많이 사용하지 않는다면 사용하는 시점에서 왜 사용했는지 알기위한 것),  
html 부품들 최상단 div (저희 학원수업은 java에 익숙하고 css,html,script 엔 덜익숙하기때문에  주석 필수입니다.  
// gnb 는 이런게 있는 겁니다.  
<div class='gnb'>
    ...... 
    ...... 
</div> 
학원에서 강사님과 예제를 풀면서 작명했던 이름이 아닌 어떠한 것들.




제일 중요한건 다같이 함수명,html클래스id명,파라미터,변수명 등등 본인이 대중적으로 이런거 만들때는 이렇게쓰더라(ex  dao.makeList(?) , div.container 등등 ) 라고 알고있는게 아니면 팀장에게 먼저 상의해주시거나 
따로 메모를 해놔줍니다. 주석으로만 처리한다고 생각하시면 안됩니다. 

log => 날짜별로 txt,img,ppt문서들을 담는다  uml,exerd 같은경우는 캡처해서 img로 따로 첨부

예를들어 8/2일 log 프론트팀 log 를 작성하게 되면,
개인 컴퓨터 바탕화면에 [802프론트] 라는 폴더를 만들고  [802프론트] 폴더 안에 802.txt 나 참고할 자료들을(img,코드파일html,css,js,jsp등등 ) 넣어서 
깃허브 log > 프론트log > [802프론트] 폴더를 드래그앤드롭 하면 됩니다.

log는 길지않아도됩니다. 다만, 내가 오늘 뭘했는지 이건 뭔지 오늘 뭘공부했는지 자기가 작성한 소스코드에 대한거를 남이 보기좋게 적는 행위입니다. 
우린 매일 서로의 코드를 전부 해독할 시간이 없을수도있지만, 로그가 있다면 서로의 코드에대해서 감을 잡을수있습니다. log 작성은  팀원들을 위한게 아닌  나를 위한 행위라고 생각하시면 됩니다.
